# 环境
1. 当前项目的python环境使用conda管理，建议使用conda创建一个新的环境来安装依赖。
2. 安装依赖：在项目根目录下运行以下命令安装所需
3. 当前项目的前端环境使用Node.js管理，建议使用nvm来管理Node.js版本。

# 背景
当前项目是为了实现一个`珠宝鉴定助手`的MVP程序, 它具备如下的能力:
1. 通过图片对一个珠宝进行粗粒度的鉴定
2. 对于鉴定结果进行结构化存储

# 资料
当前项目所有的开发资料和设计文档均放在`docs` 目录下，开发者可以在该目录下找到相关的资料和文档。

# 工作逻辑
1. 先指定计划, 对于有争议的点点, 或者不清晰的点, 经过Human-in-loop讨论清楚后开发.
2. 通过`docs`目录下的roadmap.md文件来查看当前项目的开发计划和进度。 如果当前没有roadmap.md文件, 可以通过查看`docs`目录下的其他文档来了解当前项目的开发计划和进度。
3. 在制定了`roadmap.md`文件后, 对于每个`路标`, 分拆成一个个的`任务`, 每个`任务`对应一个`issue`, 通过`issue`来跟踪每个`任务`的进度和状态。每个路标使用独立的文件来跟踪开发过程.
4. 每完成一个`任务`, 需要在`路标`对应的文件中记录下完成的内容和相关的资料
5. 每完成一个`路标`, 需要在`roadmap.md`文件中更新该路标的状态和相关的资料
6. 及时补充测试和执行git commit, 让每个commit包含一个完整的功能点, 以便于后续的代码回滚和问题排查.
7. 及时更新`docs`目录下的相关文档
