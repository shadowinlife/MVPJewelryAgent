## 0. v0.3.1 更新范围

本版本不是 UI 重写，而是小修版。

UI-Spec_v0.3 的页面结构、视觉风格、端形态和核心交互保持不变。v0.3.1 仅补充以下内容：

1. 将对应产品文档从 Product-Spec_v0.3 同步为 Product-Spec_v0.4。
2. 增加“功能真实状态标识”，区分真实功能、Mock 功能、占位功能和后续功能。
3. 增强管理后台的数据导出、OSS 文件访问状态、AI/OCR 接入状态和交付验收入口。
4. 强化用户端与后台对“当前功能是否已真实接入”的展示规则，避免把漂亮页面误认为完整业务系统。
5. 不新增复杂视觉设计，不改变曜齐 Atelier YAOQI 的品牌调性。

## 3. 功能真实状态标识

### 3.1 增加原因

Product-Spec_v0.4 明确区分：

- UI 原型 MVP
- Mock 流程 MVP
- 业务闭环 MVP
- 可公测生产 MVP

因此 UI 必须能让项目方和工程师清楚看到每个模块当前状态，避免出现“页面看起来完成，但后端、数据库、OSS、AI、OCR 均未真实接入”的误判。

### 3.2 状态类型

所有关键功能模块建议使用以下四类状态：

#### 真实可用

说明：该功能已经接入真实后端、数据库或第三方服务，并通过基础测试。

显示建议：

- 标签文案：真实可用
- 颜色：成功 / 低风险 #4F7A5B

#### Mock 演示

说明：该功能页面和流程可走通，但数据为模拟数据，尚未接入真实服务。

显示建议：

- 标签文案：Mock 演示
- 颜色：警告 / 中风险 #B8860B

#### 占位待接入

说明：该功能只保留入口或按钮，点击后提示后续接入。

显示建议：

- 标签文案：待接入
- 颜色：中性灰 #B9B0A3

#### 后续版本

说明：该功能不属于 MVP P0 范围，只作为路线图展示。

显示建议：

- 标签文案：后续版本
- 颜色：浅暖灰 #DDD2C2

### 3.3 必须标识状态的模块

用户端必须标识：

- 图片上传：真实 OSS 上传 / Mock 上传 / 本地预览
- 证书 OCR：真实 OCR / Mock OCR / 手动录入
- AI 报告：真实 API 生成 / Mock 报告 / 待接入
- 登录：真实手机号验证码 / Mock 登录 / 微信待接入
- 会员权限：真实后端权限 / 前端演示锁定
- 案例库：真实数据库 / Mock 案例
- 客户简洁版报告：真实裁剪 / Mock 文案

管理端必须标识：

- 用户管理：真实数据库 / Mock 用户
- 会员开通：真实写入 / 前端演示
- 案例管理：真实案例 / Mock 案例
- 历史案例导入：真实入库 / 解析预览 / 占位
- 知识文件管理：真实上传 / 本地 Mock
- 数据导出：真实导出 / 待接入
- AI/OCR 失败记录：真实日志 / 待接入

### 3.4 展示位置

用户端：

- 普通用户不应看到过多工程状态。
- 内测期间可在页面底部或设置页显示“当前为内测版本，部分功能为演示数据”。
- 不建议在正式用户界面到处显示工程标签，避免降低信任感。

管理端：

- 必须在管理工作台显示功能接入状态。
- 每个模块顶部显示当前状态。
- 后台表格中的 Mock 数据必须明显标识，避免误导运营判断。

## 7. 新建案例页

### 7.1 页面目标

让用户尽量少思考地完成资料上传和基础信息录入。

### 7.2 推荐流程

采用分步表单：

#### 第 1 步：选择任务

- 品类
- 用途
- 来源渠道

用途包括：

- 购买
- 出售
- 回收
- 法拍
- 学习
- 直播选品
- 客户咨询
- 商业选品

#### 第 2 步：上传图片

- 珠宝自然光图
- 打灯图
- 证书图
- 上手图
- 背面图
- 瑕疵图
- 法拍截图
- 卖家文案截图

#### 第 3 步：OCR 确认

- 证书图片预览
- OCR 字段列表
- 低置信度提示
- 手动修改
- 跳过 OCR
- 当前 OCR 接入状态，内测可见

#### 第 4 步：补充信息

- 尺寸
- 重量
- 圈口
- 珠径
- 叫价 / 起拍价
- 成交价
- 证书机构
- 证书编号
- 卖家文案
- 用户备注
- 是否有出售 / 回收 / 寄售意向
- 用户心理价，选填

#### 第 5 步：生成报告

- 保存草稿
- 生成报告
- 生成前显示费用/次数提示，若启用配额
- 如果 AI 未真实接入，必须在内测环境明确提示“当前为 Mock 报告”

### 7.3 移动端要求

- 每一步字段不要太多。
- 底部固定“下一步 / 保存草稿”按钮。
- 图片上传要支持手机拍照和相册。
- 上传成功后立即显示水印预览图。
- 原图不直接展示永久链接。

## 10. 会员锁定模块

### 10.1 出现位置

- 具体价格带
- 回收价
- 法拍上限
- 压价策略
- 转售建议
- 相似历史案例

### 10.2 锁定卡文案示例

标题：价格带与回收价为高级内容  
说明：升级后可查看合理入手价、流通成交价、即时回收价和转售建议。

辅助文案：AI 估价仅供参考，不替代线下复检和正式评估。

### 10.3 UI 要求

- 不显示具体价格数字。
- 不显示被打码但可猜测的数字长度。
- 不把高级内容放在前端后再隐藏。
- 卡片采用淡金色边框。
- 不做倒计时和强促销。
- 点击后进入会员权益页或联系管理员说明。

## 12. 图片预览与水印

### 12.1 用户端图片规则

- 默认展示水印预览图。
- 不展示原图下载按钮。
- 图片加载使用短时效签名 URL。
- 图片过期后自动重新请求。
- 长按保存可不专门支持，浏览器层面的保存无法完全阻止，但水印能降低盗用价值。

### 12.2 水印样式

建议：

- 低透明度
- 斜向或角落重复
- 不遮挡关键鉴定区域
- 内容包括 YAOQI、案例编号、用户尾号或时间

### 12.3 管理端图片规则

- 管理员可查看原图。
- 管理员查看原图需要记录日志。
- 原图下载按钮可放 P1，不建议 P0 开放给普通管理员以外角色。
- OSS 文件访问状态应在后台可见，包括是否为私有、是否使用签名链接、签名有效期。

## 16. 文案规则

### 16.1 必须避免

- 绝对保真
- 官方鉴定
- 权威认证
- 100% 准确
- 一定升值
- 保证回收
- 包赚钱

### 16.2 推荐文案

- “AI 辅助判断，仅供参考”
- “建议结合线下复检”
- “当前图片不足以支持强结论”
- “该价格区间为参考，不构成交易承诺”
- “高级价格内容需开通对应会员后查看”
- “客户简洁版不包含内部回收价和压价策略”
- “当前为内测版本，部分功能可能为演示数据”
- “该模块尚未接入真实服务，请以后台接入状态为准”

## 17. v0.3.1 UI 验收标准

### 17.1 用户端

- 手机浏览器能完成登录。
- 手机浏览器能新建案例。
- 手机浏览器能上传图片和证书。
- OCR 结果可确认和修改。
- 报告能按会员权限展示。
- 高级价格内容被锁定且不泄漏。
- 客户简洁版不能生成分享链接。
- 图片预览带水印。
- 内测环境下，Mock 报告和真实报告不得混淆。

### 17.2 管理后台

- 管理员能登录后台。
- 管理员能查看用户。
- 管理员能手动开通会员。
- 管理员能查看案例。
- 管理员能导入 Markdown 案例。
- 管理员能上传知识文件。
- 管理员能查看 OCR/AI 失败记录。
- 管理员能导出用户、案例和高价值线索。
- 管理员能查看功能接入状态。
- 管理员能区分真实数据和 Mock 数据。

### 17.3 安全 UI

- 普通用户看不到原图下载入口。
- 普通用户看不到 OSS 永久链接。
- 高级报告内容不能只靠前端隐藏。
- 删除、重新生成、导出敏感内容都有确认。
- 所有报告页显示免责声明。
- 后台导出和原图查看必须有操作确认和日志入口。
